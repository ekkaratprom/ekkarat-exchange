01

PCI DSS คืออะไร

Foundations — The Standard

PCI DSS คือมาตรฐานความปลอดภัยและข้อกำหนดทางธุรกิจที่ออกแบบมาเพื่อให้องค์กรที่ประมวลผล จัดเก็บ หรือส่งข้อมูลบัตรชำระเงิน รักษาสภาพแวดล้อมที่ปลอดภัย

🏛️

ก่อตั้งโดย 5 แบรนด์หลัก

  • Visa
  • MasterCard
  • American Express
  • Discover Financial Services
  • Japan Credit Bureau (JCB)
📜

ประวัติและเวอร์ชัน

  • สร้างขึ้นในปี 2004
  • เวอร์ชันก่อนหน้า: 3.2.1
  • เวอร์ชันปัจจุบัน: 4.0
  • บังคับใช้เต็มรูปแบบ: เมษายน 2024
  • ดูแลโดย PCI Security Council

ประโยชน์ vs ผลกระทบหากไม่ปฏิบัติตาม

✓ ปฏิบัติตาม

  • ระบบปลอดภัย ลูกค้าไว้วางใจ
  • ลดความเสี่ยงข้อมูลรั่วไหล
  • เพิ่มชื่อเสียงกับธนาคารและผู้ออกบัตร
  • เป็นพันธมิตรทางธุรกิจที่น่าเชื่อถือ

✗ ไม่ปฏิบัติตาม

  • เสียชื่อเสียง สูญเสียลูกค้า
  • ถูกฟ้องร้อง เรียกค่าเสียหาย
  • ถูกปรับจากผู้ออกบัตร
  • ถูกปรับจากรัฐบาล
  • อาจถูกยกเลิกสมาชิกภาพการรับบัตร
02

ข้อมูลที่ต้องปกป้อง

Cardholder Data vs Sensitive Authentication Data

PCI แบ่งข้อมูลออกเป็น 2 ประเภทหลัก — บางอย่างเก็บได้ บางอย่างห้ามเก็บเด็ดขาด

ประเภท ข้อมูลที่รวมอยู่ เก็บได้หรือไม่
Cardholder Data (CD)
ข้อมูลผู้ถือบัตร		 • Primary Account Number (PAN) — เลข 16 หลัก
• Cardholder Name — ชื่อผู้ถือบัตร
• Expiration Date — วันหมดอายุ
• Service Code — รหัสบริการ 		เก็บได้
หากมีเหตุผลและมาตรการที่เข้มงวด
Sensitive Authentication Data (SAD)
ข้อมูลยืนยันตัวตน		 • Full Magnetic Stripe — แถบแม่เหล็กทั้งหมด
• CSC / CVV2 / CID / CAV2 — รหัสความปลอดภัย
• PIN และ PIN Block 		ห้ามเก็บ
ใช้ได้เฉพาะตอนทำธุรกรรม
⚠ จดจำ

ข้อมูล SAD ห้ามจัดเก็บในรูปแบบใดๆ ทั้งสิ้น — ไม่ว่าจะเป็นกระดาษ ไฟล์ ฐานข้อมูล หรืออีเมล ใช้ได้เฉพาะ ณ เวลาทำธุรกรรมเท่านั้น แล้วต้องลบทิ้งทันที

วัตถุประสงค์ของ CVV/CSC

CVV (Card Verification Value) หรือ CSC (Card Security Code) มีจุดประสงค์เดียวคือ ยืนยันว่าผู้ทำธุรกรรมมีบัตรอยู่ในครอบครองจริง โดยเฉพาะธุรกรรมที่ไม่ได้สแกนบัตร (Card Not Present) เช่น การซื้อของออนไลน์หรือทางโทรศัพท์

03

6 เป้าหมายของ PCI

The Six Goals & Twelve Requirements

องค์กรต้องปฏิบัติตาม ข้อกำหนด 12 ข้อ ที่ครอบคลุม 6 เป้าหมายหลัก (Goals) โดยต้องกรอก Self-Assessment Questionnaire (SAQ) และ Attestation of Compliance (AOC) ส่งให้ acquiring bank

01

Build & Maintain a Secure Network

ติดตั้ง firewall, การตั้งค่าระบบอย่างปลอดภัย ไม่ใช้ค่า default จากผู้ผลิต

02

Protect Cardholder Data

ปกป้องข้อมูลที่จัดเก็บ และเข้ารหัสข้อมูลขณะส่งผ่านเครือข่ายสาธารณะ

03

Vulnerability Management

ใช้ anti-virus, พัฒนาและดูแลระบบและซอฟต์แวร์ให้ปลอดภัยอยู่เสมอ

04

Strong Access Control

จำกัดการเข้าถึงตามความจำเป็น, ใช้ Unique ID และคุมการเข้าถึงทางกายภาพ

05

Monitor & Test Networks

ติดตามการเข้าถึงทรัพยากรเครือข่าย ทดสอบระบบและกระบวนการอย่างสม่ำเสมอ

06

Information Security Policy

กำหนดนโยบายความปลอดภัยข้อมูลที่ครอบคลุมทุกฝ่ายในองค์กร

Acquiring Bank คือธนาคารที่ประมวลผลการชำระเงินบัตรในนามของร้านค้า ธนาคารกลุ่มนี้ กำหนดให้ร้านค้าทุกรายต้องปฏิบัติตาม PCI DSS

— ข้อกำหนดของระบบการชำระเงิน
04

การจัดเก็บและป้องกัน

Storage & Protection Methods

"Stored" หมายถึง ทุกที่, ทุกรูปแบบ — ฐานข้อมูล, อีเมล, คอมพิวเตอร์, มือถือ, เอกสารกระดาษ, ใบเสร็จบัตร — ทั้งหมดต้องปลอดภัย

01
Encryption

เข้ารหัสข้อมูล ทำให้อ่านไม่ออกหากไม่มี encryption key

02
Tokenization

แทนที่ PAN จริงด้วยโทเค็นอิเล็กทรอนิกส์ที่ไม่มีความหมาย

03
Truncation

ปกปิดเลขบัตร ตัดออกอย่างน้อย 6 หลัก แสดงเฉพาะ 6 แรก/4 ท้าย

รูปแบบ PAN ที่ถูกและผิด

ตัวอย่าง: หมายเลขบัตรจริง 1234 5678 4321 8765

1234 56XX XXXX 8765 VALID
1234 5678 4321 8765 FULL — INVALID
1234 5678 4321 XXXX ปกปิดน้อยเกินไป
05

รหัสผ่านที่แข็งแกร่ง

Your First Line of Defense

23 ล้านบัญชีถูกแฮก โดยใช้รหัสผ่านเดียวกัน — 123456

ข้อกำหนดของ PCI DSS

ข้อกำหนดขั้นต่ำ

  • อย่างน้อย 7 ตัวอักษร
  • ต้องมีทั้งตัวเลขและตัวอักษร
  • เปลี่ยนทุก 90 วัน
  • ห้ามซ้ำกับรหัสเดิม
  • ห้ามใช้ค่า default จากผู้ผลิต
  • ล็อกบัญชีหลังป้อนผิด ≤ 6 ครั้ง

Best Practice

  • ผสม upper/lower/ตัวเลข/อักขระพิเศษ
  • ความยาว 8-10 ตัวขึ้นไป
  • ไม่ใช้วลีหรือเลขที่นิยม
  • ไม่ใช้ข้อมูลส่วนตัว (ชื่อ ทีม ลูก)
  • ไม่แชร์กับใคร แม้แต่ครอบครัว
  • ไม่จด ไม่ส่งทางอีเมล

ลืมรหัสผ่าน?

  3. ติดต่อ IT หรือหัวหน้างานเพื่อขอรีเซ็ตรหัสอย่างถูกต้อง
  4. เปลี่ยนรหัสใหม่ทันทีหลังจากได้รับรหัสชั่วคราว
06

Unique ID

Use Your Own — Never Share

One person = One ID. ทุกการกระทำต้องสืบย้อนได้ถึงบุคคลคนเดียว ไม่มีข้อยกเว้น

ID ของคุณคือ ลายเซ็นดิจิทัล ทุกครั้งที่มีการเข้าใช้ระบบด้วย ID ของคุณ ระบบจะถือว่าเป็นคุณที่ทำ — ไม่ว่าจริงๆ จะเป็นคุณหรือไม่ก็ตาม

— หลักการ Accountability ใน PCI DSS

❌ การแชร์ Credentials

  • ตรวจสอบย้อนหลังไม่ได้
  • ระบบ logging ใช้ไม่ได้ผล
  • ความรับผิดตกอยู่ที่คุณ
  • เสี่ยงโดนลงโทษทางวินัย

✓ Unique ID

  • ตรวจสอบ (audit) ได้ชัดเจน
  • ระบุตัวผู้กระทำได้
  • ปฏิบัติตาม PCI DSS
  • ปกป้องตัวเองและทีม

Physical Security

25% ของเหตุการณ์ปลอดภัยมาจากการขโมยทางกายภาพ

"มีไฟร์วอลล์ระดับเทพ ใช้แอนตี้ไวรัสตัวท็อป รหัสผ่านแข็งแกร่งทุกคน แต่ถ้าทิ้งประตูหน้าเปิดไว้ ทุกอย่างก็ไร้ความหมาย

ภัยคุกคามทางกายภาพ

💳
Card Skimming

ติด skimmer บนเครื่องรูดบัตรพกพา เพื่อขโมยข้อมูลบัตรลูกค้า

🔧
Device Tampering

การแอบดัดแปลงอุปกรณ์เพื่อเก็บหรือส่งข้อมูลออก

📦
Device Theft

ขโมยอุปกรณ์ที่มีข้อมูลผู้ถือบัตรไปทั้งเครื่อง

🗑️
Dumpster Diving

รื้อค้นถังขยะหาเอกสารหรือใบเสร็จที่มีข้อมูลบัตร

มาตรการป้องกัน

01
🔒 Lock Devices

ล็อกคอมพิวเตอร์ ลิ้นชัก ตู้เก็บเอกสารเมื่อไม่ใช้

02
👀 Inspect Terminals

ตรวจเครื่องรูดบัตรประจำ สังเกตร่องรอยการดัดแปลง

03
🚪 Access Control

จำกัดผู้เข้าออกพื้นที่ที่มีอุปกรณ์ประมวลผลข้อมูล

04
👥 Escort Visitors

บุคคลภายนอกต้องมีผู้ดูแลตลอดเวลา

05
🗑️ Shred Documents

ทำลายเอกสารด้วยเครื่องย่อยกระดาษมาตรฐาน PCI

06
📋 Inventory

บันทึกรายการอุปกรณ์ครบ ตรวจนับสม่ำเสมอ

08

Malware

ซอฟต์แวร์ร้าย — ภัยที่มองไม่เห็น

Malware คือซอฟต์แวร์ที่ออกแบบมาเพื่อบุกรุกและทำลายคอมพิวเตอร์ โดยไม่ได้รับความยินยอม

🔒
Ransomware

เข้ารหัสไฟล์เหยื่อ เรียกค่าไถ่ผ่าน Bitcoin เพื่อปลดล็อก

👁️
Spyware

แอบสอดแนม ขโมยข้อมูลการใช้งานและข้อมูลส่วนตัว

📢
Adware

แสดงโฆษณาที่ไม่ต้องการ เปลี่ยนเส้นทางเบราว์เซอร์

🦠
Viruses

แก้ไขไฟล์ปกติให้กลายเป็นพาหะ — 10% ของการติดมัลแวร์

🐴
Trojans

ปลอมเป็นโปรแกรมปกติ มักแฝงผ่านอีเมลและเว็บที่ติดเชื้อ

6 วิธีป้องกัน

  1. ระวังอีเมลแปลก ๆอย่าเปิดอีเมล ไฟล์แนบ หรือคลิกลิงก์จากผู้ส่งที่ไม่รู้จัก
  2. หลีกเลี่ยงเว็บไซต์น่าสงสัยโดยเฉพาะเว็บที่โฆษณาในสแปม
  3. ระวัง USB Drive ที่ไม่รู้จักห้ามเสียบ USB ที่ไม่รู้ที่มา
  4. อัปเดตระบบสม่ำเสมอติดตั้ง security update โดยเร็วที่สุด
  5. ใช้ Anti-virusต้องเปิดและอัปเดตก่อนออนไลน์เสมอ
  6. ระวัง Social Mediaโฆษณาปลอมและแอปอันตรายแพร่กระจายผ่านโซเชียล

สัญญาณว่าติดเชื้อ

🚨 สังเกตเหล่านี้

  • พฤติกรรมแปลกตอนเปิดเครื่อง/เบราว์เซอร์
  • เห็นโปรแกรมหรือไฟล์ที่ไม่เคยติดตั้ง
  • เครื่องช้าลงผิดปกติแม้ reboot
  • Anti-virus ถูกปิดโดยไม่ทราบสาเหตุ
  • เครื่องค้าง/crash บ่อย
  • ป๊อปอัพโฆษณาเด้งเยอะ

🛠️ เมื่อสงสัย

  • หยุดใช้งานทันที
  • ไม่กรอกข้อมูลใดๆ เพิ่ม
  • ติดต่อแผนก IT
  • อย่าพยายามแก้ไขเอง
  • แจ้งหัวหน้างานทราบ
09

Email Security

อีเมลไม่ปลอดภัยโดยธรรมชาติ

Email ส่งข้อมูลแบบ ไม่เข้ารหัส โดยปริยาย แฮกเกอร์ใช้โปรแกรม "sniffer" ดักจับและอ่านได้ระหว่างเดินทาง

📧 ห้ามทำ

ห้ามส่งรหัสผ่าน, PAN เต็ม, CVV, หรือข้อมูลละเอียดอ่อนใด ๆ ผ่านอีเมล ไม่ว่ากรณีใด — แม้ลูกค้าจะร้องขอก็ตาม

อันตรายของอีเมลฟรี

บริการเช่น Hotmail, Gmail ไม่เข้ารหัสและไม่ปลอดภัยสำหรับข้อมูลธุรกิจ คุณไม่รู้ว่าข้อมูลถูกเก็บที่ไหน นานเท่าไร และมีมาตรการอะไร — แม้ลบอีเมลแล้ว ข้อมูลก็ยังอยู่บนเซิร์ฟเวอร์

แนวทางความปลอดภัย

✓ ควรทำ

  • เขียนอีเมลด้วยความระมัดระวัง
  • ตรวจชื่อผู้รับก่อนกด Send
  • ใช้โทรศัพท์เมื่อต้องส่งข้อมูลลับ
  • ใช้ระบบที่เข้ารหัสสำหรับรหัสผ่าน

✗ ห้ามทำ

  • ส่งรหัสผ่าน/ข้อมูลบัตรทางอีเมล
  • คลิกลิงก์จากผู้ส่งที่ไม่รู้จัก
  • ใช้อีเมลฟรีส่งข้อมูลบริษัท
  • ใช้อีเมลบริษัทส่งเรื่องส่วนตัว
  • ส่งข้อมูลผ่าน WiFi สาธารณะ

DO & DON'T

แนวปฏิบัติประจำวัน — Daily Practice

✓ DO — ทำสิ่งเหล่านี้

01
Secure Receipts

เก็บใบเสร็จในตู้ล็อก ทำลายเมื่อไม่จำเป็นด้วย shredder

02
Protect the PAN

ตรวจสอบใบเสร็จให้แสดงแบบ truncated เสมอ

03
Keep It Tidy

รักษาความเป็นระเบียบ ใช้นโยบาย Clean Desk

04

Quiz Answers & Explanations

7 ข้อสอบจากบทเรียนทั้งหมด พร้อมเฉลยและคำอธิบายอย่างละเอียด

QUESTION 01 Physical Security

เช้าวันหนึ่งเมื่อมาถึงที่ทำงาน คุณพบว่าเพื่อนร่วมงานจดข้อมูลบัตรเครดิตของลูกค้าลงบนกระดาษและทิ้งไว้หลังเคาน์เตอร์ คุณควรทำอย่างไร?

  • ทิ้งกระดาษโน้ตลงในถังขยะธรรมดา
  • ทำลายกระดาษโน้ตด้วยเครื่องย่อยกระดาษที่ได้มาตรฐาน PCI
✓ Correct Answer · Explanation

ต้องใช้ PCI-compliant shredder เท่านั้น เพราะถังขยะธรรมดาเสี่ยงต่อ Dumpster Diving — อาชญากรอาจรื้อค้นถังขยะหาข้อมูล ทั้งยังควรแจ้งเพื่อนร่วมงานและรายงานเหตุการณ์ให้หัวหน้าทราบ

QUESTION 02 Email Security

ต้องส่งรหัสผ่านบัญชีใหม่ให้พนักงานที่ทำงานจากบ้าน ควรส่งรหัสผ่านอย่างไร?

  • ใส่ในอีเมลฉบับเดียวกับลิงก์
  • ส่งในอีเมลแยกฉบับ
  • โทรหาพนักงานและบอกรหัสผ่านทางโทรศัพท์
✓ Correct Answer · Explanation

อีเมล ไม่ปลอดภัยโดยธรรมชาติ — ส่งข้อมูลแบบไม่เข้ารหัส แฮกเกอร์สามารถดักจับได้ด้วยโปรแกรม sniffer ห้ามส่งข้อมูลลับเช่นรหัสผ่านทางอีเมลโดยเด็ดขาด ไม่ว่าจะแยกอีเมลหรือไม่ก็ตาม

QUESTION 03 Best Practice

ข้อใดเป็นแนวปฏิบัติด้านความปลอดภัยที่ดีที่สุดในการจัดการข้อมูลบัตรชำระเงิน?

  • จัดเก็บข้อมูลบัตรในสถานที่ปลอดภัย
  • จดเลข CVV ไว้เพื่อจำได้
  • ส่ง PAN ทางอีเมลหากมีการร้องขอ
  • ทำสำเนาเลข PAN เก็บไว้
✓ Correct Answer · Explanation

PCI DSS อนุญาตให้จัดเก็บ Cardholder Data ได้ หากมีเหตุผลทางธุรกิจและมาตรการความปลอดภัยที่เข้มงวด ส่วน CVV ห้ามจดเด็ดขาด (เป็น SAD), PAN ห้ามส่งทางอีเมลแบบเต็ม, และไม่ควรทำสำเนาที่เพิ่มความเสี่ยง

QUESTION 04 Truncation

หมายเลขบัตรเครดิตของลูกค้าคือ 1234 5678 4321 8765 ใบเสร็จควรแสดงหมายเลขนี้อย่างไร?

  • 1234 5678 4321 8765
  • xxx xxxx 4321 8765
  • xxxx xxxx xxxx 8765
  • 1234 5678 4321 xxxx
✓ Correct Answer · Explanation

หลัก Truncation ของ PCI — ต้องปกปิดอย่างน้อย 6 หลัก แสดงได้เฉพาะ 6 หลักแรก และ/หรือ 4 หลักสุดท้าย เท่านั้น ตัวเลือกที่ปลอดภัยที่สุดคือแสดงเพียง 4 หลักสุดท้าย ปกปิดอีก 12 หลัก

QUESTION 05 CVV / CSC

จุดประสงค์หลักของ CSC (CVV) บนบัตรชำระเงินคืออะไร?

  • ยืนยันว่าบัตรยังไม่หมดอายุ
  • ยืนยันว่าผู้ถือบัตรมีบัตรอยู่ในครอบครอง
  • ยืนยันว่าผู้ถือบัตรมีเงินเพียงพอสำหรับธุรกรรม
✓ Correct Answer · Explanation

CVV/CSC คือเลข 3-4 หลักที่ ไม่ได้พิมพ์บนแถบแม่เหล็ก/ชิป และ ห้ามจัดเก็บ — ออกแบบมาเพื่อพิสูจน์ว่าผู้ทำธุรกรรมถือบัตรจริงในมือ (สำคัญมากในการซื้อออนไลน์/โทรศัพท์) ส่วนวันหมดอายุใช้ Expiration Date และยอดเงินคงเหลือเป็นหน้าที่ของธนาคารผู้ออกบัตร

QUESTION 06 Strong Password

ข้อใดถือเป็นรหัสผ่านที่แข็งแกร่ง?

  • 901246723
  • manutd
  • admin
  • momP!anc3!!
✓ Correct Answer · Explanation

momP!anc3!! ผ่านทุกข้อกำหนด — ยาว 11 ตัว, มีตัวอักษร+ตัวเลข+อักขระพิเศษ, ผสมพิมพ์ใหญ่-เล็ก ส่วนตัวเลือกอื่น: 901246723 มีแต่ตัวเลข, manutd สั้นและเป็นชื่อทีมที่เดาง่าย, admin เป็น default password ที่อันตรายที่สุด

QUESTION 07 Password Management

จำรหัสผ่านไม่ได้ ควรทำอย่างไร?

  • ใช้บัญชีของเพื่อนร่วมงานแทน
  • ไม่เป็นไร เพราะจดไว้บนกระดาษ post-it แล้ว
  • ขอให้หัวหน้างานหรือแผนก IT รีเซ็ตรหัสผ่านให้
  • เดารหัสผ่าน
✓ Correct Answer · Explanation

ติดต่อ IT หรือหัวหน้างาน เป็นวิธีที่ปลอดภัยและถูกต้องตาม PCI DSS เพราะมีการยืนยันตัวตนและบันทึก audit trail ส่วนตัวเลือกอื่น: ใช้บัญชีคนอื่น ละเมิด Unique ID, จดบน post-it ใครก็เห็นได้, เดารหัส ระบบจะล็อกหลังป้อนผิด 6 ครั้ง

การรู้คำตอบที่ถูกต้องคือก้าวแรก แต่การ เข้าใจเหตุผลเบื้องหลังต่างหากที่ทำให้คุณเป็นพนักงานที่ปลอดภัยจริงๆ
