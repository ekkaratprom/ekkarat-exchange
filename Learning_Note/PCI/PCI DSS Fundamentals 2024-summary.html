§ 00

Introduction to PCI DSS

PCI DSS (Payment Card Industry Data Security Standard) คือมาตรฐานความปลอดภัยสำหรับธุรกิจที่ ประมวลผล / เก็บ / ส่ง ข้อมูลบัตรชำระเงิน (เครดิต + เดบิต)

สร้างปี 2004 โดย 5 แบรนด์ใหญ่ (the "five brands"): Visa MasterCard American Express Discover JCB

ประโยชน์ของการ Comply

  • ลูกค้าเชื่อใจ — ระบบปลอดภัย
  • ลดความเสี่ยงข้อมูลรั่ว ทั้งวันนี้และอนาคต
  • รักษาความสัมพันธ์กับธนาคาร / payment brands

ถ้าไม่ Comply จะเจออะไร?

  • ชื่อเสียงพัง เสียลูกค้า
  • ถูกฟ้องร้อง โดนปรับจากผู้ออกบัตรและรัฐ
  • ถูกยกเลิกสิทธิ์รับบัตร

PCI DSS v4.0 — เวอร์ชันล่าสุด

ออกเดือนเมษายน 2024 อัปเดตเพื่อรับมือภัยใหม่ๆ — บังคับ MFA, กฎรหัสผ่านที่เข้มขึ้น, ป้องกัน Phishing & Social Engineering ดีขึ้น

§ 01

Cardholder Data vs SAD

ข้อมูลที่ PCI DSS ปกป้องแบ่งเป็น 2 ประเภทหลัก

✓ เก็บได้ (อย่างปลอดภัย)

Cardholder Data (CD)

  • PAN — เลข 16 หลักหน้าบัตร
  • Cardholder Name — ชื่อเจ้าของบัตร
  • Expiration Date — วันหมดอายุ
  • Service Code
✗ ห้ามเก็บเด็ดขาด!

Sensitive Auth Data (SAD)

  • Full Magnetic Stripe
  • CVV / CSC / CID / CAV2
  • PIN / PIN Block
เคล็ดจำ: ของที่ เห็นได้ด้วยตา (หน้าบัตร + ชื่อ) = CD เก็บได้ / ของที่ ใช้ยืนยันตอนจ่าย (CVV, PIN, magnetic stripe) = SAD ห้ามเก็บ

"Store" หมายถึงเก็บที่ไหนบ้าง?

ทุกที่ ทุกรูปแบบ — Database, อีเมล, คอม, มือถือ, กระดาษ (ใบเสร็จก็นับ!)

3 วิธีปกป้อง Stored Cardholder Data

วิธีทำอะไร
Encryptionเข้ารหัสข้อมูล อ่านไม่ได้ถ้าไม่มี key
Tokenizationแทน PAN ด้วย token อิเล็กทรอนิกส์
Truncationปิดบัง PAN เหลือ 4 ตัวแรกหรือ 6 ตัวท้าย (เช่น xxxx xxxx xxxx 1234)
§ 02

Goal 1 — Build & Maintain Secure Network

1
Requirement 1

Network Security Controls

ติดตั้ง + ดูแล firewall, router ป้องกัน CDE จากภายนอก

ครอบคลุม virtual devices, cloud, container ด้วย

2
Requirement 2

Secure Configurations

เปลี่ยน default password, ลบ software/account ที่ไม่ใช้, ปิด service ที่ไม่จำเป็น

Mindset
"Req 1 = กำแพงเมือง · Req 2 = ล็อกประตูทุกบาน"
§ 03

Goal 2 — Protect Account Data

3
Requirement 3

Protect Stored Data

Data at Rest — เก็บเฉพาะที่จำเป็น, ห้ามเก็บ SAD, ใช้ encryption/truncation/tokenization

4
Requirement 4

Protect Data in Transit

Data in Transit — เข้ารหัสตอนส่งผ่าน public network, ปกป้อง wireless

เปรียบเทียบ: Req 3 = ใส่ข้อมูลใน "ตู้เซฟ" / Req 4 = ใส่ใน "รถหุ้มเกราะ"
§ 04

Goal 3 — Vulnerability Management

5
Requirement 5

Protect from Malware

Anti-virus, anti-phishing training, update + patch ตลอด

6
Requirement 6

Secure Systems & Software

Secure coding, จัดลำดับ vulnerability, WAF, change management

Vulnerability Management คืออะไร?

กระบวนการ ค้นหา → ประเมิน → แก้ไข → รายงาน จุดอ่อนในระบบอย่างต่อเนื่อง — เพราะภัยใหม่เกิดขึ้นทุกวัน

§ 05

Goal 4 — Access Control

7
Requirement 7

Need to Know

Least Privilege — ให้สิทธิ์เท่าที่งานต้องใช้ Deny by default

8
Requirement 8

Unique ID + MFA

1 คน 1 ID — accountability + traceability พร้อม MFA และ password เข้มแข็ง

9
Requirement 9

Physical Access

Badge reader, visitors ต้องมีคนพา, ทำลายข้อมูลอย่างปลอดภัย, ตรวจ POS ป้องกัน tampering

3 Authentication Factors: Something you know (รหัส) + Something you have (โทรศัพท์/token) + Something you are (ลายนิ้วมือ) → รวม 2+ = MFA

§ 07

Goal 6 — Security Policy

12
Requirement 12

Information Security Policy

Acceptable use policy, ทำ risk analysis ปีละครั้ง, security training (รับเข้า + ทุกปี), screen พนักงาน, Incident Response Plan, ตรวจสอบ third-party

§ 08

Password Security

PCI Requirements สำหรับ Password

กฎรายละเอียด
ความยาวอย่างน้อย 12 ตัว (v4.0) / 7 ตัว (v3.2.1)
ส่วนผสมต้องมีทั้ง ตัวเลข + ตัวอักษร
เปลี่ยนทุก 90 วัน
ห้ามซ้ำต่างจากของเก่า
Defaultห้ามใช้ default ของผู้ผลิต
Lockoutล็อก account หลังพยายามผิดไม่เกิน 6 ครั้ง

Top 10 รหัสผ่านที่โดนแฮกบ่อยที่สุด

123456 · 123456789 · qwerty · password · 111111 · 12345678 · abc123 · 1234567 · password1 · 12345

😱 ข้อเท็จจริง: รหัส 123456 เพียงตัวเดียว — บัญชีโดนแฮก 23 ล้านบัญชี ในปีเดียว

Best Practice — Don't Make It Easy

  • ผสม upper/lower/เลข/สัญลักษณ์ ยาวอย่างน้อย 12 ตัว
  • ไม่ใช้ข้อมูลส่วนตัวที่หาได้บน social media
  • ไม่ใช้ทริคสลับตัวอักษรง่ายๆ (เช่น P@ssw0rd)
  • ห้าม share — ห้ามจด — ห้ามส่งทางอีเมล
Be Yourself
"1 คน · 1 Unique ID" — ห้าม share credentials เด็ดขาด เพราะ accountability!
§ 09

Malware & Phishing

ประเภทของ Malware

ประเภททำอะไรระดับ
Ransomwareเข้ารหัสไฟล์ เรียกค่าไถ่ (Bitcoin)🔴🔴🔴 สูงมาก
Spywareแอบเก็บข้อมูล / รหัส / keylog🔴🔴 สูง
Virusฝังในไฟล์ปกติ แพร่ผ่านการเปิดไฟล์🔴🔴 สูง
Trojanปลอมเป็นโปรแกรมดีๆ (มักปลอมเป็น antivirus)🔴🔴 สูง
Adwareโฆษณากวนใจ + redirect เว็บ🔴 น้อย-กลาง

สัญญาณว่าเครื่องติด Malware

  • กิจกรรมแปลกๆ ตอนเปิดเครื่อง/browser
  • มีโปรแกรม/ไฟล์แปลกปลอม
  • เครื่องช้าลง รีสตาร์ทแล้วก็ยังช้า
  • Antivirus ถูกปิดเอง เปิดกลับไม่ได้
  • เครื่อง crash บ่อย
  • Popup โฆษณาเพิ่มขึ้นผิดปกติ

Phishing — การหลอกออนไลน์

ปลอมตัวเป็นคน/องค์กรที่ไว้ใจ เพื่อขโมยรหัส บัญชี เงิน

🚩 Red Flags ของ Phishing

  • ผู้ส่งน่าสงสัย (โดเมนแปลกๆ)
  • คำทักทายทั่วไป "Dear Customer"
  • สร้างความเร่งด่วน / ขู่
  • Link / Attachment น่าสงสัย
  • สะกดผิด ไวยากรณ์แย่
  • ขอข้อมูลส่วนตัว/รหัส
กฎทอง: ธนาคาร / Tech company / องค์กรจริง — ไม่เคยขอรหัสหรือข้อมูลส่วนตัวทางอีเมล!
Phishing Response
"STOP → THINK → VERIFY" — หยุด · คิด · ตรวจสอบ
§ 10

Payment Card Do's & Don'ts

✓ Do's

สิ่งที่ควรทำ

  • เก็บใบเสร็จใน locked storage
  • แสดง PAN แบบ truncated (xxxx xxxx xxxx 1234)
  • โต๊ะทำงาน เรียบร้อย (Clean Desk)
  • อ่าน + เข้าใจ security policy
✗ Don'ts

สิ่งที่ห้ามทำ

  • ทิ้งเอกสารในที่ไม่ปลอดภัย
  • ทิ้งใบเสร็จลงถังขยะปกติ (ใช้ shredder)
  • เขียน/เก็บ CVV เด็ดขาด!
  • ส่ง PAN เต็มทางอีเมล (ต้อง encrypted/truncated)
⚠️ จำให้แม่น: CVV / CSC / PIN = SAD = ห้ามเก็บในรูปแบบใดๆ ทั้งสิ้น ไม่ว่าจะเขียน, save, email, screenshot หรืออะไรก็ตาม
§ ★

Quiz + เฉลย

🎯 ทดสอบความเข้าใจของคุณ — คลิกตัวเลือกที่คิดว่าถูก แล้วกด "เฉลย" เพื่อดูคำตอบ

Question 01 / 05
According to PCI, vulnerability scanning should be performed:
  • Yearly
  • Quarterly
  • Half-yearly
  • Monthly
✓ คำตอบที่ถูกต้อง
B. Quarterly (ทุกไตรมาส)
ตาม PCI DSS Requirement 11 (Goal 5) — ต้องทำ vulnerability scan ทั้งภายในและภายนอก ทุก 3 เดือน เพื่อค้นหาช่องโหว่ใหม่ๆ ส่วน penetration test ทำปีละครั้ง

💡 ทริคจำ: "V-Q, P-Y" — Vuln scan = Quarterly, Pen test = Yearly
Question 02 / 05
An information security policy should apply to:
  • Employees only
  • Contractors only
  • Employees and contractors
  • Managers only
✓ คำตอบที่ถูกต้อง
C. Employees and contractors
ตาม PCI DSS Requirement 12 — คำว่า "Personnel" ครอบคลุมทุกคนที่มีส่วนเกี่ยวข้องกับการปกป้องข้อมูล ทั้งพนักงาน full-time, part-time, temporary, contractors, consultants และ third parties

💡 ถ้ายกเว้นใครคนหนึ่ง = สร้างจุดอ่อนในระบบ
Question 03 / 05
Which of the following best describes PCI 4.0's "customized approach" to compliance?
  • The customized approach allows you to choose which requirements are best suited to your organization
  • The customized approach allows you to substitute any of the 12 requirements for customized requirements that suit your organization
  • The customized approach allows you to substitute your own control to meet the objective of any PCI DSS requirement
✓ คำตอบที่ถูกต้อง
C. Use your own control to meet the objective
Customized Approach เป็นฟีเจอร์ใหม่ใน v4.0 — คุณสามารถใช้ control ของตัวเอง แทน control มาตรฐาน แต่ต้องบรรลุ objective เดียวกัน

⚠️ ไม่ใช่การ "เลือก" requirement หรือ "เปลี่ยน" requirement — ต้องทำครบทั้ง 12 ข้อเสมอ เพียงแค่ "วิธีการ" ที่ยืดหยุ่นได้
Question 04 / 05
What is meant by the term 'Vulnerability Management'?
  • Managing information system's log files
  • Ensuring your company's premises is physically secure
  • Finding weaknesses in your company's network infrastructure
  • Scanning emails for viruses
✓ คำตอบที่ถูกต้อง
C. Finding weaknesses in network infrastructure
Vulnerability Management = กระบวนการ ค้นหา → ประเมิน → แก้ไข → รายงาน จุดอ่อนในระบบอย่างต่อเนื่อง ตรงกับ Goal 3 (Req 5, 6) และ Req 11 ของ PCI DSS

💡 ตัวเลือกอื่นก็เป็นเรื่อง security เหมือนกัน แต่เป็นคนละหมวด: A = Log Management, B = Physical Security, D = Anti-malware
Question 05 / 05 (เลือกได้หลายข้อ)
Which of the following is considered cardholder data?
  • PAN
  • Expiry date
  • Magnetic stripe data
  • Cardholder name
✓ คำตอบที่ถูกต้อง
A. PAN · B. Expiry date · D. Cardholder name
Cardholder Data (CD) ที่เก็บได้ (อย่างปลอดภัย) = PAN, ชื่อ, วันหมดอายุ, service code

❌ Magnetic Stripe Data = SAD (Sensitive Authentication Data) — ห้ามเก็บเด็ดขาด หลัง authorize เช่นเดียวกับ CVV และ PIN

💡 ทริคจำ: ของที่ เห็นได้ด้วยตาหน้าบัตร = CD / ของที่ ซ่อนอยู่ (stripe, chip, CVV) = SAD
